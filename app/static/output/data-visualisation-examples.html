Data Visualisation Examples

Date Sat 26 September 2020 By James Dey Category Solution Architecture. PDF Tags data visualisation
Data visualisation can cover:-
  • Reports - e.g. bar charts
  • Dashboards - a combination of reports & text that provides a visual information front end for an enterprise
  • Diagrams - e.g. data flow diagrams
Listed below are a some data visualisation options that you can adopt to enhance your website

Amcharts

AmCharts is an example of a data visualisation reports library (based on D3.js). With this, you'll need to add the data content and amend the javascript as suits your needs.
  • Pros: Offers a lot of flexibility, You remain in control of your data
  • Cons: Need to be technically savvy, doesn't provide a WYSIWYG editor, doesn't provide as many chart types as more expensive tools, doesn't render well on a mobile
amCharts

Creately

Creately is an example of a data visualisation diagrammer, similar to Visio. This comes in 2 parts:-
  • You model your diagram on the Creately website
  • You either supplied javascript code in your website or attach an image
  • Pros: Creately, offer a WYSIWYG editor with lots of visualisations, you don't need to be as tech savvy, intelligent device rendering
  • Cons: Unless you buy a plan, your diagram is public

Tableau

Tableau is a data visualisation report & dashboard tool which is used by everybody from small companies & individuals through to global multinationals. Tableau allows you to connect to a large variety of relational & non-relational data sources and has a wide range of visualisation themes, filters, and coding logic that you can apply
  • Pros: It provides a freemium model to get you started. It's a true enterprise tool
  • Cons: Tableau Online is expensive & Tableau Server very expensive.
Tableau comes in 2 main parts:- * Tableau Desktop - This is where you connect & register your data sources with Tableau and design your reports & dashboards. This is downloadable software. * Tableau Server/Online/Public - These are 3 server side components Tableau Server is an On-Prem server installation. Using this, you can create folders for your reports, dashboards & data sources and strictly apply security to users & groups. You can also schedule the regeneration of data sources, reports & dashboards.
  • Pros: If you are very concerned about data security, then you might argue having an On-prem server is the way to go
  • Cons: Tableau Server is more expensive to license than Tableau Online and you have to pay for your own infrastructure & staff
Tableau Online is a cloud version of Tableau Server
  • Pros: If you want a modern enterprise level data visualisation reporting & dashboard tool, this is the way to go.
  • Cons: For commercial use, it's expensive. Your reports, dashboards & data sources are reliant on Tableau. Despite the cost, their support is community based in the main.
Tableau Public is the free version of Tableau Online which allows you to deploy your reports, dashboards & data sources to the cloud & share the data visualisation with other people
  • Pros: If you've not got sensitive data, Tableau Public is fine, and acts a stepping stone to using Tableau Online
  • Cons: Anybody can see your data. You never know if Tableau will pull the plug on Tableau Public at some point.
